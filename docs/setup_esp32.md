# ESP32 Setup Guide — SentinelEdge Firmware

## Prerequisites
- PlatformIO (VSCode extension or CLI)
- USB cable (data, not charge-only)
- ESP32 Dev Board (38-pin)
- All sensors wired per pin spec

---

## Wiring

```
MPU6050:
  VCC  → 3.3V
  GND  → GND
  SDA  → GPIO21
  SCL  → GPIO22

DHT11:
  VCC  → 3.3V
  DATA → GPIO4
  GND  → GND

MQ5:
  VCC  → 5V (needs 5V for heater!)
  AOUT → GPIO34
  GND  → GND
  NOTE: 3-5 minutes warmup required after power-on

NEO-6M GPS:
  VCC  → 3.3V
  GND  → GND
  TX   → GPIO16   (ESP32 RX2)
  RX   → GPIO17   (ESP32 TX2)
```

---

## Phase 1 First: Run ML Pipeline

Before flashing inference firmware, you must have run the full ML pipeline to generate the headers:

```
firmware/src/model/fault_model.h      ← TFLite model as C array
firmware/src/model/model_settings.h   ← quant params + scaler params
```

These are generated by `ml/04_convert_to_tflite.py`. If missing, the firmware won't compile.

## Configure WiFi and MQTT

Edit `firmware/src/main.cpp`:

```cpp
static const char* WIFI_SSID     = "YOUR_WIFI_SSID";
static const char* WIFI_PASSWORD = "YOUR_WIFI_PASSWORD";
static const char* MQTT_HOST     = "192.168.1.100";  // Pi's IP address
```

## Flash: Inference Mode

```bash
cd sentineledge/firmware
pio run -e inference --target upload
pio device monitor --baud 115200
```

Key numbers to record:

- `infer=Xms` — TFLite inference latency
- `total=Xms` — full pipeline latency (target < 100ms)

## Flash: Data Collection Mode

Use this before you have a model — just prints raw sensor CSV.

```bash
pio run -e collection --target upload
pio device monitor --baud 115200 > data/raw_session.txt
```

Output format:
```
# ax,ay,az,gx,gy,gz
-0.123456,0.234567,9.814321,0.002341,-0.001234,0.000234
...
```

Then use `ml/serial_logger.py` for automated collection with labels.

## Flash: Drift Check Mode

Verifies that C++ feature extraction matches Python within 1%:

```bash
pio run -e drift_check --target upload
pio device monitor --baud 115200
```

In serial monitor, send `d` — ESP32 prints:
```
DRIFT_CLASS:0
9.81000042
6.93401241
...  (42 lines)
DRIFT_CLASS:1
...
```

Save to file and compare:
```bash
python ml/06_feature_drift_validation.py --compare esp32_drift.txt
```

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| AllocateTensors() failed | Tensor arena too small | Increase `kTensorArenaSize` in model_settings.h |
| MPU6050 not found | Wrong I2C pins or VCC issue | Check wiring GPIO21/22, ensure 3.3V not 5V |
| MQTT not connecting | Wrong IP or Mosquitto not running | `ping <pi-ip>` from PC; check `sudo systemctl status mosquitto` |
| Inference >100ms | Arena too large causing cache misses | Reduce arena, rebuild |
| GPS no fix (indoor) | No satellite view | Expected indoors; use last cached coordinates |
